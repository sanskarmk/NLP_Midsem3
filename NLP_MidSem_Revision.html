1 · Introduction to NLP Application / Scenario

NLP makes computers understand and generate human language. It is a hard problem mainly because language is ambiguous, context-dependent, and full of world knowledge that isn't written down.

The 4 main ambiguity types

  • Lexical — a word has multiple meanings/POS. "book" = noun (a book) or verb (to book a ticket); "bank" = river bank / money bank.
  • Syntactic / structural — sentence has multiple parse trees. "I saw the man with the telescope" — who has the telescope?
  • Semantic — multiple meanings even after parsing. "The chicken is ready to eat" — chicken eats, or chicken is eaten?
  • Pragmatic / referential — meaning depends on context/intent; pronoun reference (anaphora). "John told Bill that he passed" — who is he?

Levels of language understanding (low → high)

  • Phonology – sounds · Morphology – word structure (un-happi-ness) · Lexical – word meaning/POS
  • Syntax – grammar/parse structure · Semantics – literal meaning · Pragmatics – meaning in context
  • Discourse – meaning across sentences (coreference). Modern LLMs reach up to the discourse level via attention.

NLP pipeline (classical preprocessing)

Sentence segmentation → Tokenization → lowercasing/normalization → stop-word removal → stemming/lemmatization → POS tagging → parsing → feature/vector representation.

Classical vs Modern vs Hybrid: classical = rules + statistics (fast, light, explainable); modern = neural/LLMs (powerful reasoning, heavy). Hybrid = classical for light preprocessing + guardrails, LLMs for advanced reasoning — usually the best real-world design.
WORKED Example 1 — Identify ambiguity & design
Q: "Visiting relatives can be boring." (a) Name the ambiguity type and explain both readings. (b) Is automatic detection an NLP task? Which understanding level resolves it?
(a) Type = Syntactic (structural) ambiguity. Two valid parse structures:
Reading 1: "[Visiting relatives] can be boring" → the act of visiting relatives is boring.
Reading 2: "Visiting relatives [can be boring]" → relatives who are visiting are boring.
(b) Yes, it's an NLP task (syntactic parsing / word-sense & structure disambiguation). It is resolved at the syntax level (choosing the correct parse tree), often helped by semantic/pragmatic context.
How you'd implement: tokenize → POS tag → generate candidate parses → rank with a probabilistic parser or LLM using context. Hybrid: rules + neural model.
SELF-CHECK Example 2 — Try it yourself
Q: For "The bank approved my loan after I sat by the river bank," (a) identify the ambiguity type for the word bank, (b) name the level that resolves it, and (c) state which preprocessing step would NOT remove the word "the".
(a) Lexical (word-sense) ambiguity — financial bank vs river bank.
(b) Resolved at the semantic / lexical level using surrounding context ("loan" vs "river").
(c) "the" is a stop-word; it is removed by stop-word removal, NOT by stemming/lemmatization or tokenization.

2 · Evaluation Metrics Numerical (easy marks)

NLP systems are judged by more than accuracy. From a confusion matrix (TP, FP, FN, TN):

Precision = TPTP + FP    Recall = TPTP + FN    Accuracy = TP + TNTP + TN + FP + FN
F1 = 2 · Precision · RecallPrecision + Recall  (harmonic mean)
  • Precision = of all predicted positive, how many were right (don't cry wolf).
  • Recall = of all actual positives, how many we caught (don't miss any).
  • F1 balances both — use when classes are imbalanced.
WORKED Example 1 — Spam classifier
Q: A spam filter gives TP = 40, FP = 10, FN = 5, TN = 45. Compute Precision, Recall, F1, Accuracy.
Precision = 40 / (40+10) = 40/50 = 0.80
Recall = 40 / (40+5) = 40/45 = 0.889
F1 = 2(0.80)(0.889)/(0.80+0.889) = 1.4222/1.6889 = 0.842
Accuracy = (40+45)/100 = 0.85
SELF-CHECK Example 2 — Try it yourself
Q: A sentiment model gives TP = 30, FP = 20, FN = 10, TN = 40. Compute Precision, Recall, F1, Accuracy.
P = 0.60 · R = 0.75 · F1 = 0.667 · Accuracy = 0.70

3 · Vector Semantics & Cosine Similarity Numerical

Core idea: represent each word/document as a vector. Words used in similar contexts get similar vectors (the distributional hypothesis), so similarity becomes math.

  • Term–document matrix: columns = documents, cells = counts → a document vector (column) or word vector (row).
  • Term–term co-occurrence matrix: how often two words appear within a ±k context window → word vectors.
  • To compare vectors you can't just subtract — use the dot product, normalized by length → cosine similarity.
a · b = Σ aibi     |a| = √(Σ ai²)
cos(a,b) = a · b|a| · |b| = Σ aibi√Σai² · √Σbi²

cos ≈ 1 → very similar (same semantic space); cos ≈ 0 → unrelated. Dividing by length makes it independent of document/vector size.

WORKED Example 1 — Cosine similarity
Q: Find the cosine similarity between word vectors A = [3, 2, 0, 5] and B = [1, 0, 4, 2].
Dot product: A·B = (3·1)+(2·0)+(0·4)+(5·2) = 3+0+0+10 = 13
|A| = √(9+4+0+25) = √38 = 6.164
|B| = √(1+0+16+4) = √21 = 4.583
cos(A,B) = 13 / (6.164 × 4.583) = 13 / 28.25 = 0.460
Interpretation: moderate similarity (not strongly related).
SELF-CHECK Example 2 — Try it yourself
Q: Compute cosine similarity between A = [1, 2, 3] and B = [2, 0, 1].
cos(A,B) = 5 / (√14 · √5) = 5 / 8.367 = 0.598
Steps to match: dot = 2+0+3 = 5, |A| = √14 = 3.742, |B| = √5 = 2.236.

4 · TF–IDF Numerical

Raw counts over-weight common words. TF–IDF boosts words that are frequent in this document but rare across the collection. (Jurafsky & Martin conventions, log base 10.)

tft,d = 1 + log10(countt,d)  (if count > 0, else 0)
idft = log10( Ndft )     tf-idft,d = tft,d × idft
  • N = total number of documents; dft = number of documents containing term t.
  • Collection frequency counts every occurrence; document frequency (df) only counts how many documents contain it.
  • A word in all documents → idf = log(N/N) = log 1 = 0 → killed off (e.g. "the", "good").
WORKED Example 1 — Compute TF–IDF
Q: Corpus has N = 3 documents. In Doc1: "cheese" appears 8 times (df=3), "sauce" 12 times (df=2), "truffle" 4 times (df=1). Compute the TF–IDF weight of each term in Doc1 (log base 10).
cheese: tf = 1+log10(8) = 1+0.903 = 1.903; idf = log10(3/3) = log10(1) = 0 → tf-idf = 0.000
sauce: tf = 1+log10(12) = 1+1.079 = 2.079; idf = log10(3/2) = 0.176 → tf-idf = 2.079×0.176 = 0.366
truffle: tf = 1+log10(4) = 1+0.602 = 1.602; idf = log10(3/1) = 0.477 → tf-idf = 1.602×0.477 = 0.764
Note how "cheese" (in every doc) is zeroed out, while the rare "truffle" gets the highest weight — exactly the goal of IDF.
SELF-CHECK Example 2 — Try it yourself
Q: Corpus has N = 4 documents. In a document, "the" appears 20 times (df=4), "player" 6 times (df=2), "Messi" 3 times (df=1). Compute each TF–IDF (log base 10).
the = 0.000 · player = 0.535 · Messi = 0.889
tf: the=2.301, player=1.778, Messi=1.477 · idf: the=0, player=log10(2)=0.301, Messi=log10(4)=0.602.

5 · Word Embeddings — Skip-Gram Negative Sampling Numerical (the hard one)

Sparse TF–IDF vectors are long and full of zeros. Word2Vec learns short dense vectors. SGNS is self-supervised: it uses the corpus itself to create labels — no human annotation.

  • Pick a target word w. Words inside the ±k window = positive context (label 1). Random words outside = negative samples (label 0), usually k per positive.
  • Treat it as binary logistic regression: predict "is this a real context word?" We don't care about the prediction — we want the learned weights = the embeddings.
  • Each word ends up with a target vector and a context vector; the final embedding combines (sums) them.
P(+|w,cpos) = σ(cpos·w) = 11 + e−(cpos·w)    P(−|w,cneg) = σ(−cneg·w) = 11 + e+(cneg·w)
Gradient updates (learning rate η):
cpos ← cpos − η[σ(cpos·w) − 1]·w
cneg ← cneg − η[σ(cneg·w) − 0]·w
w ← w − η{ [σ(cpos·w)−1]cpos + Σ[σ(cneg·w)]cneg }
σ is the sigmoid. "Predicted − actual": for the positive word actual = 1, for negatives actual = 0. No derivation needed in the exam — just plug into these three update lines.
WORKED Example 1 — One SGNS iteration
Q: Target w = [0.2, 0.3], positive context c+ = [0.4, 0.1], two negatives cn1 = [−0.2, 0.3], cn2 = [0.1, −0.4], η = 0.1. Do one update.
Step 1 – dot products. w·c+ = 0.08+0.03 = 0.11; w·cn1 = −0.04+0.09 = 0.05; w·cn2 = 0.02−0.12 = −0.10
Step 2 – sigmoids. σ(0.11)=0.5275; σ(0.05)=0.5125; σ(−0.10)=0.4750
Step 3 – errors. positive: σ−1 = 0.5275−1 = −0.4725; negatives use σ−0 = 0.5125 and 0.4750
Step 4 – update c+: c+ − 0.1(−0.4725)[0.2,0.3] = [0.4,0.1] + [0.00945,0.0142] = [0.4095, 0.1142]
Update cn1: [−0.2,0.3] − 0.1(0.5125)[0.2,0.3] = [−0.2102, 0.2846]
Update cn2: [0.1,−0.4] − 0.1(0.4750)[0.2,0.3] = [0.0905, −0.4143]
Update w: gradient = (−0.4725)[0.4,0.1] + 0.5125[−0.2,0.3] + 0.4750[0.1,−0.4] = [−0.1890,−0.0473]+[−0.1025,0.1538]+[0.0475,−0.1900] = [−0.2440,−0.0835].
w ← [0.2,0.3] − 0.1(−0.2440,−0.0835) = [0.2244, 0.3084]
SELF-CHECK Example 2 — Try it yourself
Q: Target w = [0.1, 0.1], positive c+ = [0.5, 0.2], one negative cn = [−0.1, 0.4], η = 0.1. Do one update of c+, cn and w.
c+ = [0.5048, 0.2048] · cn = [−0.1051, 0.3949] · w = [0.1292, 0.0894]
Checks: w·c+=0.07 → σ=0.5175; w·cn=0.03 → σ=0.5075. Positive error = −0.4825, negative error = 0.5075.

6 · CBOW & GloVe Conceptual (minor)

Both are alternative ways to learn dense word vectors. The professor said: no full numerical on CBOW; GloVe is conceptual with possibly a minor question.

  • CBOW (Continuous Bag of Words): the mirror image of skip-gram — uses the surrounding context words to predict the target/center word. Skip-gram does the opposite (center → context). CBOW is faster and good for frequent words; skip-gram is better for rare words and small data.
  • GloVe (Global Vectors): a count-based + prediction hybrid. It factorizes the global co-occurrence matrix so the dot product of two word vectors ≈ log of their co-occurrence count. It captures global statistics (whole corpus at once), unlike Word2Vec which slides locally window-by-window.
  • One-hot encoding: vector of size = vocabulary, a single 1 at the word's index. Carries no similarity info; used only as the input that gets multiplied by the embedding matrix to "look up" the dense vector.
WORKED Example 1 — CBOW vs Skip-gram
Q: For the sentence "the cat sat on the mat" with center word "sat" and window ±2, state the (input → output) training pair for (a) CBOW and (b) Skip-gram.
Context window of "sat" (±2) = {the, cat, on, the}.
(a) CBOW: input = context {the, cat, on, the} → output = "sat" (predict center from context).
(b) Skip-gram: input = "sat" → outputs = each context word: (sat→the), (sat→cat), (sat→on), (sat→the).
SELF-CHECK Example 2 — Try it yourself
Q: (a) Which model (CBOW / skip-gram) would you prefer for a small corpus with many rare technical terms? (b) What does GloVe's objective make the dot product of two word vectors approximate?
(a) Skip-gram (better for rare words / small data). (b) The log of the words' co-occurrence count.

7 · N-gram Language Modeling Numerical

A language model assigns a probability to a word sequence / predicts the next word. By the chain rule: P(w₁…wₙ) = ∏ P(wᵢ | w₁…wᵢ₋₁). That needs impossibly long histories, so we make the Markov assumption: a word depends only on the previous 1 (bigram) or 2 (trigram) words.

Bigram: P(wₙ | wₙ₋₁) = count(wₙ₋₁, wₙ)count(wₙ₋₁)     Trigram: P(wₙ | wₙ₋₂, wₙ₋₁) = count(wₙ₋₂, wₙ₋₁, wₙ)count(wₙ₋₂, wₙ₋₁)
Sentence: P(s) = ∏ P(wᵢ | wᵢ₋₁)   (use log-sum for long sentences to avoid underflow)
Convention: add sentence boundaries <s> and </s> if the question says so. In the count table the row = first/previous word, column = next word.
WORKED Example 1 — Bigram probabilities & sentence probability
Q: Training corpus (with boundaries):
<s> I am Sam </s>
<s> Sam I am </s>
<s> I like Sam </s>
Find P(I|<s>), P(am|I), P(Sam|am), P(</s>|Sam), then P(<s> I am Sam </s>).
Unigram counts: <s>=3, I=3, am=2, Sam=3, like=1, </s>=3.
P(I|<s>) = count(<s> I)/count(<s>) = 2/3 = 0.667
P(am|I) = count(I am)/count(I) = 2/3 = 0.667
P(Sam|am) = count(am Sam)/count(am) = 1/2 = 0.500
P(</s>|Sam) = count(Sam </s>)/count(Sam) = 2/3 = 0.667
Sentence: P = P(I|<s>)·P(am|I)·P(Sam|am)·P(</s>|Sam) = 0.667×0.667×0.5×0.667 = 0.1481
SELF-CHECK Example 2 — Try it yourself
Q: Corpus:
<s> the cat runs </s>
<s> the dog runs </s>
<s> the cat sleeps </s>
Compute P(<s> the cat runs </s>) using bigrams.
P = 1.000 × 0.667 × 0.500 × 1.000 = 0.333
P(the|<s>)=3/3=1, P(cat|the)=2/3, P(runs|cat)=1/2, P(</s>|runs)=2/2=1.

8 · Smoothing, Interpolation & Backoff Numerical / Conceptual

Problem: an unseen bigram gets probability 0, which zeroes the whole sentence. Fixes:

Laplace (Add-1): P*(wₙ | wₙ₋₁) = count(wₙ₋₁, wₙ) + 1count(wₙ₋₁) + V  (V = vocabulary size)
Interpolation: P̂ = λ₁P(wₙ|wₙ₋₂,wₙ₋₁) + λ₂P(wₙ|wₙ₋₁) + λ₃P(wₙ),   Σλ = 1
Backoff: use trigram; if unseen → back off to bigram; if unseen → unigram.
  • Laplace is simple but manipulates the counts a lot (steals too much mass) — used mainly for classification, not big LMs.
  • Interpolation always mixes all orders (higher weight λ to trigram). Backoff only drops to a lower order when the higher one is missing. These are preferred in practice.
WORKED Example 1 — Add-1 smoothing
Q: Using the same corpus as §7 (I am Sam …), vocabulary V = 6 (<s>, </s>, I, am, Sam, like). Compute the smoothed P*(am|I) and P*(Sam|I). Recall count(I)=3, count(I am)=2, count(I Sam)=0.
P*(am|I) = (count(I am)+1)/(count(I)+V) = (2+1)/(3+6) = 3/9 = 0.333
P*(Sam|I) = (0+1)/(3+6) = 1/9 = 0.111
Note the unseen "I Sam" now gets a non-zero 0.111 instead of 0 — that's the whole point.
SELF-CHECK Example 2 — Try it yourself
Q: With V = 6, count(am) = 2, count(am Sam) = 1, count(am I) = 0: compute add-1 smoothed P*(Sam|am) and P*(I|am).
P*(Sam|am) = (1+1)/(2+6) = 0.250 · P*(I|am) = (0+1)/(2+6) = 0.125

9 · Perplexity Numerical

Perplexity is the standard intrinsic evaluation of a language model — the inverse probability of the test set, normalized by the number of words. Lower perplexity = better model. Intuitively it's the model's average "branching factor" (how confused it is about the next word).

PP(W) = P(w₁w₂…w_N)−1/N = ᴺ√ ( 1P(w₁…w_N) )   =   ᴺ√ ∏ 1P(wᵢ | history)

N = number of words in the sequence. Use the same per-word probabilities your LM (bigram/trigram/neural) produced.

WORKED Example 1 — Compute perplexity
Q: A model assigns these per-word probabilities to a 4-word sentence: 0.25, 0.5, 0.1, 0.2. Find the perplexity.
Step 1 – sequence probability: P = 0.25 × 0.5 × 0.1 × 0.2 = 0.0025
Step 2 – invert & take Nth root: N = 4, PP = (0.0025)−1/4 = (1/0.0025)1/4 = (400)0.25
Step 3: 4000.25 = 4.47
Interpretation: the model is, on average, as uncertain as if choosing uniformly among ~4.5 words.
SELF-CHECK Example 2 — Try it yourself
Q: A model gives per-word probabilities 0.2, 0.4, 0.5 to a 3-word sentence. Find the perplexity.
PP = (0.04)−1/3 = (25)1/3 = 2.924
Product = 0.2×0.4×0.5 = 0.04; invert → 25; cube root → 2.924.

10 · Neural Language Models & LLMs Application + small problem

A neural LM applies a feed-forward neural network to predict the next word. Input = word embeddings of the previous words (not raw tokens), passed through hidden layer(s); the output layer uses softmax because predicting the next word is a multi-class problem over the whole vocabulary.

Forward pass: h = activation(W·x + b);   output = softmax(U·h)
Softmax: P(yᵢ) = ezᵢΣⱼ ezⱼ   (turns scores into a probability distribution that sums to 1)
  • Sigmoid at output → binary task (e.g. sentiment +/−). Softmax → multi-class (language modeling).
  • Sentence embedding = combine word vectors by element-wise mean or max.
  • Pros vs n-gram: handles unseen words via embeddings, captures similarity, generalizes better. Cons: needs more data/compute, less interpretable.
  • LLMs = neural LMs with huge parameters + training data (transformers). Pre-train then fine-tune for a task/domain. Prompting: zero-shot (no examples), few-shot (a few examples). Generative AI = generate new text/image/video/code from a prompt.
WORKED Example 1 — Softmax over next-word scores
Q: A neural LM's output layer produces scores (logits) z = [2.0, 1.0, 0.1] for the three candidate next words [dog, cat, runs]. Apply softmax to get probabilities.
Exponentiate: e²·⁰=7.389, e¹·⁰=2.718, e⁰·¹=1.105
Sum: 7.389 + 2.718 + 1.105 = 11.213
Divide: dog = 7.389/11.213 = 0.659, cat = 2.718/11.213 = 0.242, runs = 1.105/11.213 = 0.099
Predicted next word = "dog" (highest probability). Probabilities sum to 1.0. ✓
SELF-CHECK Example 2 — Try it yourself
Q: (a) Apply softmax to logits z = [1.0, 3.0, 0.5]. (b) Which output activation does a neural language model use, and why?
(a) softmax = [0.111, 0.821, 0.067] — 2nd word wins. (b) Softmax, because next-word prediction is multi-class over the whole vocabulary.
e¹=2.718, e³=20.086, e⁰·⁵=1.649; sum=24.453.

11 · POS Tagging — HMM & Viterbi Numerical (~5 marks)

Note: the professor skipped re-explaining POS tagging in this recap only because it was covered the same afternoon — but it is on the exam with a numerical (HMM + Viterbi), so it's fully included here.

An HMM models tagging with two probability tables:

  • Transition P(tagᵢ | tagᵢ₋₁) — how likely one tag follows another (rows = previous tag).
  • Emission P(word | tag) — how likely a tag produces a word.

The Viterbi algorithm uses dynamic programming to find the single most likely tag sequence.

v₁(t) = P(t | <s>) · P(word₁ | t)
vᵢ(t) = maxt' [ vᵢ₋₁(t') · P(t | t') ] · P(wordᵢ | t)    (keep a backpointer to the best t')

MEMM (Maximum-Entropy Markov Model) is a discriminative alternative that conditions on features of the words directly — a small concept question is possible.

WORKED Example 1 — Viterbi for "fish sleep"
Q: Tags = {N, V}. Initial P(N|<s>)=0.8, P(V|<s>)=0.2.
Transition: P(N|N)=0.4, P(V|N)=0.6, P(N|V)=0.7, P(V|V)=0.3.
Emission: P(fish|N)=0.5, P(fish|V)=0.2, P(sleep|N)=0.1, P(sleep|V)=0.4.
Find the most likely tag sequence for "fish sleep".
Step 1 — word "fish":
v₁(N) = P(N|<s>)·P(fish|N) = 0.8×0.5 = 0.40
v₁(V) = P(V|<s>)·P(fish|V) = 0.2×0.2 = 0.04
Step 2 — word "sleep":
v₂(N) = max[ v₁(N)·P(N|N), v₁(V)·P(N|V) ] · P(sleep|N) = max[0.40×0.4, 0.04×0.7]×0.1 = max[0.16, 0.028]×0.1 = 0.16×0.1 = 0.016  (from N)
v₂(V) = max[ v₁(N)·P(V|N), v₁(V)·P(V|V) ] · P(sleep|V) = max[0.40×0.6, 0.04×0.3]×0.4 = max[0.24, 0.012]×0.4 = 0.24×0.4 = 0.096  (from N)
Step 3 — best final: max(v₂(N)=0.016, v₂(V)=0.096) = 0.096 → V. Backpointer of V points to N.
Best tag sequence: fish/N   sleep/V (probability 0.096).
SELF-CHECK Example 2 — Try it yourself
Q: Tags = {D, N}. Initial P(D|<s>)=0.6, P(N|<s>)=0.4.
Transition: P(D|D)=0.1, P(N|D)=0.9, P(D|N)=0.5, P(N|N)=0.5.
Emission: P(the|D)=0.7, P(the|N)=0.1, P(dog|D)=0.05, P(dog|N)=0.6.
Find the best tag sequence for "the dog".
Best sequence: the/D   dog/N   (probability 0.2268)
v₁(D)=0.6×0.7=0.42, v₁(N)=0.4×0.1=0.04. v₂(N)=max[0.42×0.9, 0.04×0.5]×0.6 = 0.378×0.6 = 0.2268 (from D); v₂(D)=max[0.42×0.1,0.04×0.5]×0.05 = 0.0021. Winner = N, backpointer → D.
